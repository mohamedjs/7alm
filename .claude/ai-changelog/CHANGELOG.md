## 2026-07-22 — Lookbook hero + navbar redesigned to McLaren-headphones reference (horizontal filmstrip)
**By:** Claude (Opus 4.8)
**Summary:** Redesigned the store homepage hero and navbar to match a McLaren-headphones showcase reference the user shared: featured products travel as a scroll-driven **horizontal filmstrip** (active product centered, neighbours dimmed at the edges) on a full-bleed stage rounded at the bottom corners, with a transparent navbar floating inside the hero's top edge — replacing the old floating-pill navbar and two-column "reel dropping from top" layout. Iterated live with the user mid-session (framed card → full-bleed; flat fixed bar → transparent hero-scoped bar; flat mobile dropdown → neumorphic bento panel). Verified live in Chrome across AR/RTL (default), EN/LTR, dark and light themes; `tsc` + `npm run build` pass. **All committed by session end** — `b22bd8b` (filmstrip hero), `846915d` (navbar + bento menu), `7afb9e5` (user's own follow-on: brand logo, featured toggle, theme-color picker refinement, and hand-tuned smaller hero image sizing — 240/280/300px caps in `HeroImageLayer`) — which also swept in the previously-uncommitted storefront work from the earlier session.
**Changes:**
- `StoreNavbar.tsx`: transparent flat header floating over the hero's top edge. It is `fixed` but **hero-scoped, not sticky** (user request): the new `useHeroNavVisible()` (in `store.hooks.ts`) keeps it visible for the whole pinned N×100vh filmstrip sequence via `#lookbook-hero` bounds, then slides it up/away; on hero-less pages it leaves after ~80px of scroll. Centered nav links are uppercase+tracked micro-labels in EN but plain compact labels in AR (letter-spacing visually breaks connected Arabic glyphs — this locale-conditional `navLabelClass` pattern is intentional, keep it). Mobile burger opens a **bento-grid neumorphic panel** (2-col grid of `neu-raised-sm` tiles, active route `neu-pressed-sm` + brand tint, odd last tile spans both columns, EN/AR segmented tray below); the bar becomes a rounded `store-glass` sheet while the panel is open. `useScrollGlass` is no longer used here.
- `HeroImageLayer.tsx`: vertical drop replaced with horizontal slide + scale + dim (x ±42/84vw, scale 1→0.5→0.4, opacity 1→0.35→0), all derived from the same `scrollYProgress` so center/edges never desync. RTL flips the travel axis via a `dir` prop.
- **Two WAAPI gotchas fixed en route** (Motion hardware-accelerates scroll-linked transforms via `Element.animate`): (1) keyframe offsets outside [0,1] throw "Offsets must be monotonically non-decreasing" — edge sections must *drop* out-of-range keyframes; (2) offsets not covering 0 and 1 get IMPLICIT keyframes from the element's underlying style — an opacity track ending mid-range at 0 silently interpolates back toward 1, resurrecting hidden layers as ghosts (this was the "four titles overlay each other" bug). Both layers now drop out-of-range offsets **and** pad tracks to explicitly cover 0/1 with held edge values (`padTo01`).
- `HeroContentLayer.tsx`: giant centered headline column became a compact bottom-start corner info block (eyebrow, name h2, clamped description, price, add-to-cart + details link). Text visibility window is `WINDOW = 0.45` of a section-step (strictly < 0.5) so adjacent sections' text can never be visible simultaneously — at most one title on screen at any scroll position, plus a small y slide-through.
- `LookbookHero.tsx` (multi-section branch): full-bleed sticky stage (whole viewport width, `rounded-b-[2rem]/[3rem]` bottom corners only — user request; earlier framed-card iteration reverted), tagged `id={LOOKBOOK_HERO_ID}` for the navbar hook; bottom-center "select product" color-dot row (per-product `theme_color`, ≥md only); bottom-end circular outline prev/next arrows (`rtl:rotate-180`, confirmed mirroring live). `ProductThumbRow` no longer used by the hero. Scroll-driven N×100vh mechanism and reduced-motion click-driven fallback unchanged.
- `store.hooks.ts`: added `LOOKBOOK_HERO_ID` + `useHeroNavVisible()` (scroll listener against the hero container's pin range).
- `dictionary.ts`: +3 keys ×2 locales — `store.hero.selectLabel`, `store.hero.prevProduct`, `store.hero.nextProduct`.
**Files touched:** `src/components/store/{StoreNavbar,LookbookHero,HeroImageLayer,HeroContentLayer}.tsx`, `src/features/store/store.hooks.ts`, `src/features/i18n/dictionary.ts`
