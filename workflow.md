## 1. Next.js App Configuration

To configure the Next.js backend to communicate with n8n and the shipping providers, you need to add the following environment variables to your `.env.local` file:

```env
# Shipping Webhook Secret (Used to secure the shipping webhook endpoint)
SHIPPING_WEBHOOK_SECRET=your_secure_random_string

# n8n Webhook Secret (Used to verify requests coming from n8n for order confirmation/cancellation)
N8N_WEBHOOK_SECRET=your_n8n_secure_secret

# n8n Order Notification Webhook URL (The webhook URL generated by the n8n 'Order Notifications' workflow)
N8N_ORDER_WEBHOOK_URL=https://your-n8n-domain.com/webhook/order-status-changed

# Default Shipping Provider
DEFAULT_SHIPPING_PROVIDER=test # Change to 'bosta' for production

# Test Shipping Delay (Optional, used for the test provider simulation in milliseconds)
TEST_SHIPPING_DELAY_MS=30000

# Your App's Public URL
NEXT_PUBLIC_APP_URL=https://your-7alm-domain.com
```

**Next Steps in Next.js:**
1. Restart your Next.js server after updating the `.env.local` file.
2. Ensure your server is publicly accessible (e.g., via Vercel, ngrok, or your VPS) so n8n and shipping providers can send webhooks to it.

## 2. Meta Developer App Configuration

You need a Meta Developer App configured for WhatsApp, Facebook Messenger, and Instagram Graph API.

### 2.1. WhatsApp Configuration
1. Go to your Meta App Dashboard.
2. Add the **WhatsApp** product.
3. In the WhatsApp -> API Setup section, note your **Phone Number ID** and **Access Token** (use a permanent system user token for production).
4. Go to WhatsApp -> Configuration and set up the Webhook:
   - **Callback URL**: Enter the Webhook URL from the n8n "WhatsApp Trigger" node (from the *Order Confirmation Handler* or *Multi-Channel AI* workflow, depending on your setup. Note: WhatsApp allows one webhook per phone number. You must use a central webhook in n8n that routes messages to different workflows).
   - **Verify Token**: Create a random string (e.g., `my_whatsapp_verify_token`). You will need to enter this same token in the n8n WhatsApp Trigger node credentials.
   - Subscribe to the `messages` webhook field.

### 2.1.1. WhatsApp Message Templates Setup
WhatsApp requires you to use pre-approved **Message Templates** if you are sending a message to a user outside the standard 24-hour customer service window (which is often the case for proactive notifications like "Order Shipped" or "Order Delivered").

**How to create and configure a template in Meta Dashboard:**
1. Go to your **WhatsApp Manager** > **Message Templates**.
2. Click **Create Template**.
3. Select **Utility** as the category, give it a name (e.g., `order_shipped`), and select the language (e.g., Arabic).
4. Build the template body using variables like `{{1}}` for dynamic content.
   *Example Body*: `مرحباً {{1}} 🚚 طلبك للمنتج {{2}} في الطريق إليك!`
5. Submit for approval (usually takes a few minutes).

**How to use the template in n8n:**
1. Open the **WhatsApp Node** in n8n.
2. Change the **Operation** from `Send` to `Send Template`.
3. In the **Template** or **Template Name** field, select or type the exact name of your approved template (e.g., `order_shipped`).
4. Select the **Language** code (e.g., `ar` for Arabic).
5. In the **Components** section (under Header, Body, or Buttons), add your dynamic variables (Text parameters) in the exact order they appear in your template (e.g., Parameter 1 = `{{ $json.customerName }}`, Parameter 2 = `{{ $json.productName }}`).

> [!NOTE]  
> The n8n JSON workflows are currently configured to use the standard `Send` operation (which works for testing within a 24-hour window). For production proactive notifications (e.g., Shipped, Delivered), you will need to update those specific nodes in n8n to `Send Template` and map your variables.

### 2.2. Facebook Page Configuration
1. Add the **Messenger** product to your Meta App.
2. Generate a **Page Access Token** for your Facebook Page.
3. In Messenger -> Settings -> Webhooks, click "Add Callback URL":
   - **Callback URL**: Enter the Webhook URL from the n8n *Multi-Channel AI* workflow's "Facebook Webhook" node (e.g., `.../webhook/facebook-webhook`).
   - **Verify Token**: Enter any string. The n8n workflow is currently set up to echo back the challenge automatically.
4. Subscribe to the following Page fields: `messages`, `messaging_postbacks`, `feed` (for comments).

### 2.3. Instagram Configuration
1. Ensure your Instagram Professional account is linked to your Facebook Page.
2. Generate an **Instagram Access Token** (often the same Page Access Token works for IG via the Graph API).
3. In Messenger -> Settings -> Instagram Webhooks, set up the Webhook:
   - **Callback URL**: Enter the Webhook URL from the n8n *Multi-Channel AI* workflow's "Instagram Webhook" node (e.g., `.../webhook/instagram-webhook`).
   - **Verify Token**: Similar to Facebook, enter any string.
4. Subscribe to the `messages` and `comments` fields.

## 3. n8n Configuration

Import the three JSON workflow files into your n8n instance.

### 3.1. General Environment Variables
Set the following global environment variables in your n8n instance (or use n8n's built-in Variables feature):
- `HALM_APP_URL`: Your Next.js app URL (e.g., `https://your-7alm-domain.com`).
- `N8N_WEBHOOK_SECRET`: Must match the `N8N_WEBHOOK_SECRET` in your Next.js `.env.local` file.

### 3.2. Order Notifications Workflow Setup
1. Open the **7alm — Order Status Notifications (WhatsApp)** workflow.
2. Open the "Order Status Changed" Webhook node and copy its **Test** and **Production** URLs. Use the production URL as the `N8N_ORDER_WEBHOOK_URL` in your Next.js `.env.local`.
3. In all WhatsApp nodes (`wa-approved-001`, `wa-confirmed-001`, etc.):
   - Set up the **WhatsApp Business Cloud API Credential** using your Meta Access Token.
   - Replace `YOUR_PHONE_NUMBER_ID` with your actual WhatsApp Phone Number ID.
4. Save and Activate the workflow.

### 3.3. Order Confirmation Handler Setup
1. Open the **7alm — WhatsApp Order Confirmation Handler** workflow.
2. Open the "WhatsApp Trigger" node:
   - Copy the Webhook URL and paste it into your Meta App's WhatsApp Webhook configuration (as the Callback URL).
   - Set the Verify Token in the node credentials to match the one you entered in Meta.
3. Ensure the "Call 7alm API" HTTP Request node is correctly using the `$env.HALM_APP_URL` and `$env.N8N_WEBHOOK_SECRET`.
4. In all WhatsApp reply nodes, ensure credentials and Phone Number ID are set.
5. Save and Activate the workflow.

### 3.4. Multi-Channel AI Workflow Setup
1. Open the **7alm — Multi-Channel AI Customer Service** workflow.
2. Update the Meta Webhooks (FB/IG/WhatsApp):
   - Copy the Facebook Webhook URL and paste it into Meta's Messenger Webhook settings.
   - Copy the Instagram Webhook URL and paste it into Meta's Instagram Webhook settings.
3. Configure the AI nodes:
   - In the "Google Gemini Chat Model" node, add your Google Gemini API Key credentials.
   - In the "7alm AI Agent" node, verify the system prompt fits your store's tone.
4. Configure API Tools:
   - Ensure the "Get Active Product" and "Create Order" tools are pointing to `$env.HALM_APP_URL`.
5. Configure the Reply Nodes:
   - **WhatsApp**: Ensure credentials and Phone Number ID are set.
   - **Facebook/Instagram (HTTP Request nodes)**: Replace `YOUR_PAGE_ACCESS_TOKEN` and `YOUR_INSTAGRAM_ACCESS_TOKEN` with your actual tokens generated from the Meta App.
6. Save and Activate the workflow.
